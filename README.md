# Server Wrapped 🎓

> A year-end Discord server retrospective — automatically scrape your server's stats and display them on a beautiful Vercel-hosted webpage.

---

## Project Structure

```
server-wrapped/
├── bot/
│   ├── bot.py           ← Discord bot (scrapes and generates stats.json)
│   ├── config.py        ← ALL your customisation goes here
│   ├── requirements.txt
│   ├── Procfile         ← For Railway hosting
│   └── .env.example
└── web/
    ├── index.html       ← The recap webpage (deploy to Vercel)
    ├── stats.json       ← Generated by the bot (commit to repo)
    └── vercel.json      ← Vercel config
```

## Step-by-Step Setup

### 1. Create a Discord Bot

1. Go to https://discord.com/developers/applications
2. Click **New Application** → give it a name (e.g. "Server Wrapped")
3. Go to **Bot** tab → click **Add Bot**
4. Under **Privileged Gateway Intents**, enable:
   - ✅ **Server Members Intent**
   - ✅ **Message Content Intent** ← critical, without this it can't read messages
5. Click **Reset Token** → copy your token → save it somewhere safe
6. Go to **OAuth2 → URL Generator**:
   - Scopes: `bot`
   - Bot Permissions: `Read Messages/View Channels` + `Read Message History` + `Send Messages`
7. Open the generated URL → add the bot to your server

### 2. Configure the Bot

Edit `bot/config.py`:
- Set `DATE_FROM` to when your server was created (e.g. `"2024-09-01"`)
- Customise `KEYWORD_BUCKETS` — add/remove keywords as needed
- Add `EXCLUDE_CHANNELS` you don't want scraped
- Update `SPOTLIGHT_NAMES` with names relevant to your server
- Configure `VOTE_CATEGORIES` to define your "most likely to" style voting buckets

### 3. Set Up Locally (to run the bot once)

```bash
cd bot
cp .env.example .env
# Edit .env and paste your bot token

pip install -r requirements.txt
python bot.py
```

In Discord, type:
```
!wrapped status   ← see which channels will be scraped
!wrapped          ← full scrape (can take 5–15 mins for large servers)
!vote categories  ← list configured vote buckets
!vote <category> @person  ← cast a vote for that category
!vote results     ← see current winners
```

The bot writes `web/stats.json` when done.

### 4. Set Up Vercel (web frontend)

1. Push this whole repo to GitHub
2. Go to https://vercel.com → **Add New Project** → import your repo
3. Set **Root Directory** to `web/`
4. Click **Deploy** — that's it!

Every time you run `!wrapped` and commit `stats.json`, Vercel auto-redeploys.

```bash
git add web/stats.json
git commit -m "chore: update wrapped stats"
git push
```

## Hosting the Bot (Optional — Railway)

If you want the bot always online (so anyone can run `!wrapped` from Discord):

1. Go to https://railway.app → **New Project → Deploy from GitHub Repo**
2. Select your repo, set **Root Directory** to `bot/`
3. Add environment variable: `DISCORD_TOKEN = your_token_here`
4. Railway reads `Procfile` and starts the bot automatically

Free tier is more than enough for this use case.

## Permissions Needed

| Permission | Why |
|---|---|
| Read Messages / View Channels | See channels |
| Read Message History | Iterate through past messages |
| Send Messages | Reply with progress updates |
| Manage Messages (optional) | Required to run `!wrapped` — limits to admins |

---

## Troubleshooting

| Problem | Fix |
|---|---|
| `DISCORD_TOKEN not set` | Make sure `.env` exists with your token |
| `Missing Permissions` | Bot needs **Read Message History** on each channel |
| `Message Content Intent` error | Enable it in the Discord dev portal under Bot → Privileged Intents |
| Stats.json shows old data | Re-run `!wrapped` and commit again |
| Vercel shows blank page | Check that `stats.json` has real data (not the placeholder) |
